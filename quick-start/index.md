# h5pack — 把网页变成 App 的最短路径

## 它是什么

h5pack 是一个 H5 → Android APK 的打包工具。你不需要学 Java/Kotlin，不需要安装 Android Studio，不需要折腾 Gradle。只需要一个配置文件和一条命令，就能把你现有的 Web 项目打包成一个可以安装、可以上架的 Android 应用。

![An image](/assets/demo.png)

## 谁在用它

- **独立开发者**：想把自己的 H5 作品发布到应用商店，但不想从头学原生开发
- **小团队 / 外包团队**：客户要求出 App，但预算和工期不支持双端原生开发
- **前端工程师**：想用熟悉的 H5 技术栈做出"像 App"的东西，还能调用相机、GPS 等设备能力

## 系统架构与核心原理

```text
┌─────────────────────────────────────────────────────────────┐
│                      h5pack 生态系统                        │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│  h5pack-core│ h5pack-native │h5pack-bridge│  h5pack-iconkit│
│   (核心引擎) │ (原生容器)    │ (通信桥梁)   │   (资源生成)     │
└───────┬─────┴───────┬─────┴───────┬─────┴─────────┬───────┘
        │             │             │               │
        └─────────────┼─────────────┼───────────────┘
                      │   协作流程   │
┌─────────────────────▼─────────────▼─────────────────────────┐
│                      打包执行流程                             │
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────┐  │
│  │  读取配置    │───▶│ 资源生成与    │───▶│  应用打包与      │   │
│  │  h5pack.json│    │   注入      │    │     输出         │   │
│  └─────────────┘    └─────────────┘    └─────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

h5pack 基于 React Native 技术架构构建，其核心工作机制如下：

- 资源打包机制：将 Web 网页作为静态资源完整打包到 Android App 中

- 容器化加载：App 通过内嵌的 WebView 组件加载打包的 Web 页面，实现 H5 应用 App 化

- 桥接通信系统：通过精心设计的 H5PackBridge 模块，建立 H5 与原生平台之间的双向通信通道

## 为什么选 h5pack

| 痛点 | h5pack 怎么解决 |
|------|----------------|
| 学原生开发成本太高 | 不用学，H5 项目直接打包 |
| Android Studio 太重 | 不用装，一条命令出包 |
| 打包流程复杂 | 一个 `h5pack.json` 配置文件搞定一切 |
| H5 调不了设备能力 | Bridge 模块提供相机、GPS、麦克风、相册等原生 API |
| 改完代码要看效果要重新打包 | Dev 模式热更新，改完秒刷新 |
